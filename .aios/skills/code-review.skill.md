# Code Review 技能

## 使用说明

在进行代码审查时，请遵循以下步骤：

### 1. 代码结构分析
- 检查模块划分是否合理
- 验证命名约定是否一致
- 评估代码可读性

### 2. 质量检查清单
- [ ] 是否有重复代码
- [ ] 是否遵循 DRY 原则
- [ ] 错误处理是否完善
- [ ] 是否有合适的注释

### 3. 安全性检查
- 检查输入验证
- 审查敏感数据处理
- 验证权限控制

### 4. 性能考虑
- 识别潜在的性能瓶颈
- 检查资源泄露风险
- 评估算法复杂度

## 输出格式

审查报告应包含：
1. **总结**: 整体代码质量评分 (1-10)
2. **优点**: 代码的亮点
3. **问题**: 发现的问题（分为严重/中等/轻微）
4. **建议**: 具体的改进建议
