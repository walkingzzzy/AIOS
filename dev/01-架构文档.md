## 二、核心组件

### 2.1 AIOS Daemon (跨平台)

Node.js 守护进程，作为系统服务运行。

| 职责 | 说明 | 跨平台实现 |
|------|------|-----------|
| 生命周期管理 | 启动、停止适配器 | Node.js 进程管理 |
| 请求路由 | 分发请求到适配器 | JSON-RPC 2.0 路由 |
| 状态管理 | 维护系统状态 | SQLite / LevelDB |
| 健康监控 | 检测组件健康状态 | 心跳检测 |

**服务配置示例**:

```javascript
// daemon.config.js
module.exports = {
  name: 'aios-daemon',
  script: './dist/daemon.js',
  
  // IPC 配置
  ipc: {
    type: 'stdio', // 或 'websocket', 'http'
    port: 9876
  },
  
  // 日志配置
  log: {
    level: 'info',
    file: '~/.aios/daemon.log'
  }
};
```

| AIOS 权限级别 | 名称 | Windows | macOS | Linux |
|--------------|------|---------|-------|-------|
| 0 | public | 无需 | 无需 | 无需 |
| 1 | low | 无需/UAC | 无需/首次弹窗 | 无需 |
| 2 | medium | UAC | TCC 授权 | PolicyKit |
| 3 | high | Administrator | Accessibility | sudo |
| 4 | critical | System | Full Disk Access | root |

**权限检查 API**:
```typescript

      break;
    case 'linux':
      // 扫描 /usr/share/applications
      break;
```

### 2.5 平台脚本桥接

利用各平台的自动化能力。

| 平台 | 脚本语言 | 用途 |
|------|---------|------|
| Windows | PowerShell | 系统设置、应用控制 |
| macOS | AppleScript / JXA | 应用控制、系统设置 |
| Linux | Shell + xdotool | 系统设置、窗口控制 |
| 全平台 | Node.js child_process | 统一封装 |

**跨平台脚本执行**:
```typescript
import { exec } from 'child_process';

function runPlatformScript(scripts: PlatformScripts): Promise<string> {
  const script = scripts[process.platform];
  if (!script) throw new Error(`Unsupported platform: ${process.platform}`);
  
  return new Promise((resolve, reject) => {
    exec(script, (error, stdout) => {
      if (error) reject(error);
      else resolve(stdout);
    });
  });
}

// 使用示例
await runPlatformScript({
  darwin: 'osascript -e "set volume output volume 50"',
  win32: 'powershell "(Get-WmiObject -Class Win32_Sound).SetVolume(50)"',
  linux: 'pactl set-sink-volume @DEFAULT_SINK@ 50%'
});
```

## 四、通信协议

### 4.1 协议基础

- **基础协议**: JSON-RPC 2.0 (全平台通用)
- **传输方式**: 
  - stdio (主要方式，最跨平台)
  - WebSocket (可选，实时通信)
  - HTTP (可选，RESTful)
  - Unix Socket (Linux/macOS)
  - Named Pipe (Windows)

### 4.2 消息结构

```typescript
// 请求消息
interface AIOSRequest {
  jsonrpc: "2.0";
  id: string;
  method: string;
  params?: Record<string, any>;
}

// 响应消息
interface AIOSResponse {
  jsonrpc: "2.0";
  id: string;
  result?: any;
  error?: {
    code: number;
    message: string;
    data?: any;
  };
}

// IPC 服务协议
interface AIOSServiceProtocol {
  invoke(request: AIOSRequest): Promise<AIOSResponse>;
  checkPermission(permission: string): Promise<boolean>;
  getAdapters(): Promise<AdapterInfo[]>;
  ping(): Promise<boolean>;
  getVersion(): Promise<string>;
}
```

## 七、安全模型

### 7.1 权限分层

```typescript
enum PermissionLevel {
  PUBLIC = 0,    // 无需权限
  LOW = 1,       // 低风险操作
  MEDIUM = 2,    // 中等风险，需用户确认
  HIGH = 3,      // 高风险，需特殊权限
  CRITICAL = 4   // 关键操作，需管理员权限
}
```

### 7.2 平台权限映射

| AIOS 权限 | Windows | macOS | Linux |
|-----------|---------|-------|-------|
| PUBLIC | - | - | - |
| LOW | - | 首次弹窗 | - |
| MEDIUM | UAC 提示 | TCC 授权 | polkit |
| HIGH | 管理员 | System Preferences | sudo |
| CRITICAL | System | Full Disk Access | root |

### 7.3 安全最佳实践

1. **最小权限原则** - 只请求必要的权限
2. **用户确认** - 高风险操作需明确用户同意
3. **审计日志** - 记录所有敏感操作
4. **沙盒隔离** - 适配器运行在独立进程中
