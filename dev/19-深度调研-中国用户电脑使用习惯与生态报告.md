## 1. 核心发现 (Executive Summary)

中国用户的 Mac 使用习惯与欧美用户存在本质差异，主要体现在 **"All-in-One 应用"**、**"移动端反向吞噬桌面端"** 以及 **"特定的国产替代生态"**。

1.  **IM 即操作系统**: 微信和飞书不仅仅是聊天工具，更是文件传输助手、办公OA入口和即时会议终端。**QQ NT 的 Electron 架构暴露了调试端口，是重大自动化机会。**
2.  **Web 版的复兴**: 由于原生 Mac 客户端功能阉割（如 B站、微信读书、小红书），大量用户倾向于使用 PWA 或浏览器网页版，这为 Playwright 自动化提供了绝佳切入点。
3.  **国产软件的“降维打击”**: **WPS AirScript** 提供了比 VBA 更适合 Web 时代的自动化能力；**剪映** 的 JSON 草稿如同“后门”，绕过 UI 直接生产视频。
4.  **效率工具的新入口**: **Raycast** 在中国开发者和极客群体中迅速取代 Spotlight，配合百度/微信/汇率插件，成为事实上的“系统级 CMD 命令行”。

## 2. 细分领域深度技术规范

### 2.1 腾讯系通讯应用 (Tencent IM Suite)

| 应用 | 架构 | 自动化可行性 | 推荐技术路径 | 风险等级 |
| :--- | :--- | :--- | :--- | :--- |
| **QQ NT** | Electron | ⭐⭐⭐⭐⭐ (极高) | **Remote Debugging Port** (`--remote-debugging-port=xxxx`)。可直接连接 Chrome DevTools Protocol 控制。 | 低 |
| **微信 (Mac)** | Native (OC/Swift) | ⭐⭐ (低) | **Passive OCR + Accessibility**。Hook 方案 (`WeChatHooks`) 存在极高封号风险，严禁商业化使用。建议采用“被动读取屏幕+模拟点击”的非侵入式方案。 | 高 |
| **企业微信** | Native/CEF | ⭐⭐⭐ (中) | **Webhook Bot**。客户端机器人接口受限严重 (20MB文件限制, 频率限制)，但在内部群管理中，Webhook 是最稳妥的官方通道。 | 低 |

#### 技术实现资源 (Phase 3 补充)
*   **QQ NT 协议参考**: `Takayama-Lily/QQNT-Protocol` (GitHub) - 提供了 QQ NT 底层协议的大量分析，有助于理解 IPC 通信结构。
*   **企业微信 SDK**: `node-wecom` 或官方 `wxwork-bot` (NPM)。

### 2.2 办公软件深度集成 (Office & WPS)

**A. WPS Office (AirScript)**
WPS Mac 版不支持 AppleScript，但全面集成了 **AirScript** (基于 JavaScript)。这是 AIOS 控制 WPS 的核心。

*   **开源代码库**: **`poboll/wps_script`** (GitHub)。该项目包含了大量 AirScript 签到、表格处理的脚本示例，是极其宝贵的 "Snippets" 来源。
*   **能力边界**: 读写单元格、操作工作表、HTTP 请求 (需配置白名单)、弹窗交互。
*   **AirScript 注入示例**:
    ```javascript
    // 自动将 A1:E1 设置为标题格式
    function formatHeader() {
      let sheet = Application.ActiveSheet;
      let range = sheet.Range("A1:E1");
      range.Interior.ColorIndex = 44; // Orange
      range.Font.Bold = true;
      range.HorizontalAlignment = xlCenter;
    }
    ```

**B. Microsoft Office 365 (Mac)**
*   **JS API**: Mac 版 Excel 支持 Office.js API，但 **不支持 COM add-ins 和 ActiveX**。
*   **VBA**: Mac 版 VBA 环境是“沙盒化”的，访问本地文件系统受限严重。
*   **结论**: 优先使用 Office JS API 开发跨平台插件，避免使用 VBA。

### 2.3 信息搜索与效率工具

**A. Raycast 生态**
Raycast 是 Mac 上最热门的效率工具。
*   **GitHub 参考**: 搜索 `raycast-extensions` 可找到大量中文插件，如 `WeChat` (非官方键盘扩展)、`Easy Dic`。
*   **AIOS 策略**: 不直接造轮子，而是编写 **Raycast Script Commands** (Bash/Node.js) 供用户调用 AIOS 的能力。

**B. 浏览器与搜索**
*   **夸克 (Quark)**: Mac 版主打“AI 助手”，支持全局快捷键截屏搜题。
*   **微信读书 Web**: 优于客户端。适合使用 Playwright (Headless Chrome) 进行“沉浸式阅读”模式的自动化翻页和笔记导出。

### 2.4 专业办公与基础库 (OA/ERP/Libs)

**A. 泛微 (Weaver) / 用友 (Yonyou)**
*   **接口优先**: e-cology 和 NC Cloud 均提供标准的 RESTful / OAuth2 接口。需关注 `appid` + `spk` 鉴权机制。
*   **自动化测试**: **Playwright** 优于 Selenium，因为其对动态加载和 Frame 的处理更现代，能更好应对 OA 系统复杂的嵌套结构。

**B. 推荐开源基础库 (Open Source Libs)**
*   **Mac OCR**: 
    1.  **Apple Vision Framework** (Native Swift): 最快，最安全，零依赖。
    2.  **`EasyOCR`** (Python): 跨平台备选方案，支持中文极佳。
*   **IM SDK**:
    1.  **Feishu**: `larksuite/node-sdk` (官方) - 完美支持所有 Open API。
    2.  **DingTalk**: `dingtalk-robot-sender` (NPM) - 轻量级机器人消息发送库。

## 3. [CN-Pack] 技术路线图 (Phase 2 & 3)

### 3.1 核心控制适配器 (CN-Adapters)

基于 `packages/daemon/src/adapters` 的现有架构，建议新建 `cn` 目录存放本土化适配器。所有适配器需继承 `BaseAdapter` 并实现 `AdapterCapability` 接口。

**目录结构**:
```text
packages/daemon/src/adapters/
├── cn/
│   ├── index.ts          # 统一导出
│   ├── QQNTAdapter.ts    # QQ 调试端口控制
│   ├── WPSAdapter.ts     # WPS AirScript 注入
│   ├── FeishuAdapter.ts  # 飞书 API 客户端
│   └── WeChatAdapter.ts  # 微信 OCR 辅助
```

**核心类定义 (Spec)**:

1.  **`QQNTAdapter` (extends BaseAdapter)**
    *   *Capabilities*: `qq_send_text`, `qq_get_messages`
    *   *Impl*: 使用 `puppeteer-core` 或 `playwright` 连接本地 Debug Port (9222)。
2.  **`WPSAdapter` (extends BaseAdapter)**
    *   *Capabilities*: `wps_run_script`, `wps_read_selection`
    *   *Impl*: 利用 `clipboard` 模块复制 JS 代码，通过 AppleScript 模拟快捷键触发 WPS 的 "运行脚本" 窗口 (参考 `poboll/wps_script`)。
3.  **`WeChatAdapter` (extends BaseAdapter)**
    *   *Capabilities*: `wechat_snapshot_ocr`
    *   *Impl*: 调用 `screencapture` 截图 + macOS Vision API 识别。
4.  **`FeishuAdapter` (extends BaseAdapter)**
    *   *Capabilities*: `feishu_send_card`, `feishu_create_doc`
    *   *Impl*: 引入 `@larksuiteoapi/node-sdk` (需在 `daemon/package.json` 添加依赖)。

| 优先级 | 目标软件 | 技术路径 | 理由 |
