#### 2.2.1 音频控制

| 功能 | 实现方式 | npm 包 |

#### 2.2.4 电源管理

| 功能 | 实现方式 | 平台命令 |
|------|---------|---------|
| 锁屏 | 系统命令 | pmset / rundll32 / loginctl |
| 休眠 | 系统命令 | pmset / powrprof / systemctl |
| 关机 | 系统命令 | shutdown |
| 重启 | 系统命令 | shutdown / systemctl |

```typescript
import { exec } from 'child_process';
import { promisify } from 'util';
const execAsync = promisify(exec);

// 跨平台锁屏
async function lockScreen() {
  const platform = process.platform;
  if (platform === 'darwin') {
    await execAsync('pmset displaysleepnow');
  } else if (platform === 'win32') {
    await execAsync('rundll32.exe user32.dll,LockWorkStation');
  } else {
    await execAsync('loginctl lock-session');
  }
}
```

#### 2.2.5 应用管理

| 功能 | 实现方式 | npm 包 |
|------|---------|--------|
| 打开应用/URL | `open()` | open |
| 进程列表 | `psList()` | ps-list |
| 关闭进程 | `process.kill()` | 内置 |

```typescript
import open from 'open';
import psList from 'ps-list';

// 打开应用
await open.openApp('Google Chrome');

// 打开 URL
await open('https://google.com');

// 获取进程列表

```

#### 2.2.6 窗口管理

| 功能 | 实现方式 | npm 包 |
|------|---------|--------|
| 窗口操作 | 键盘快捷键模拟 | @nut-tree/nut-js |
| 屏幕信息 | `screen.width()` / `height()` | @nut-tree/nut-js |

```typescript
import { keyboard, Key } from '@nut-tree/nut-js';

// 窗口左半屏 (Windows: Win+Left)
await keyboard.pressKey(Key.LeftSuper, Key.Left);

// 最大化 (Windows: Win+Up)
await keyboard.pressKey(Key.LeftSuper, Key.Up);
await keyboard.releaseKey(Key.LeftSuper, Key.Up);
```

```typescript
// 层配置数据结构
interface LayerConfig {
    baseUrl: string;    // API 地址
    apiKey: string;     // API Key
    model: string;      // 选择的模型
}

// 模型信息
interface ModelInfo {
    id: string;         // 模型 ID
    name?: string;      // 显示名称
    created?: number;   // 创建时间
}

// 获取模型列表响应
interface FetchModelsResponse {
    success: boolean;
    models?: ModelInfo[];
    error?: string;
}

// 测试连接响应
interface TestConnectionResponse {
    success: boolean;
    response?: string;  // AI 响应内容
    error?: string;
}
```

**交互流程**:

1. 用户点击预设按钮 → 自动填充 API 地址

3. 点击"获取模型列表" → 调用 fetchModels API
4. 成功后模型输入框变为下拉选择
5. 用户选择模型 → 点击"测试连接"
6. 测试成功显示 AI 响应 → 点击"保存配置"

### 5.2 技术栈

| 组件 | 技术 | 理由 |
|------|------|------|
| 运行时 | Node.js 20+ | 跨平台、生态丰富 |
| 语言 | TypeScript | 类型安全 |
| 包管理 | pnpm | 高效、workspace 支持 |
| 进程通信 | stdio / WebSocket | 跨平台 IPC |
| GUI 框架 | Electron | 跨平台桌面应用 |
| AI 引擎 | 三层云端 API | 成本优化 |
| 通信协议 | JSON-RPC 2.0 | 与 MCP 兼容 |
| 测试框架 | Vitest | 快速、TypeScript 原生 |

## 六、风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| npm 包平台兼容性 | 部分功能不可用 | 提供降级方案 |
| 系统命令差异 | 跨平台行为不一致 | 平台检测 + 分支处理 |
| Electron 体积大 | 安装包过大 | 按需加载、代码分割 |
| AI 响应延迟 | 用户等待 | 流式响应、本地缓存 |
