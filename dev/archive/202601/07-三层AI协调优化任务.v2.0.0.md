## P0 - 高优先级（1-2 天）

### P0-1: 统一直达匹配入口

**问题**: `MessageRouter` 和 `TaskOrchestrator` 都实现了直达匹配逻辑，存在重复和响应格式不一致。

**涉及文件**:
- `src/core/MessageRouter.ts`
- `src/core/TaskOrchestrator.ts`

**现状**:
```typescript
// MessageRouter.ts - 返回 tier: "direct"
if (!hasScreenshot) {
  const directMatch = intentClassifier.matchDirectTool(message);
  if (directMatch) {
    // ... 直接执行 ...
    return { tier: 'direct', ... };
  }
}

// TaskOrchestrator.ts - 返回 tier: 'fast'
if (analysis.directToolCall) {
  return await this.executeDirectTool(analysis.directToolCall);
  // 内部返回 tier: 'fast'
}
```

**修复方案**:
1. `MessageRouter.smartChat` 移除直达匹配逻辑，统一走 `TaskOrchestrator.process()`
2. 在 `TaskOrchestrator` 中新增 `tier: 'direct'` 类型，区分本地直达和 Fast 层 AI 调用

```typescript
// MessageRouter.ts - 简化为纯转发
async handleSmartChat(request: JSONRPCRequest): Promise<JSONRPCResponse> {
  const { message, hasScreenshot } = request.params;
  
  const context: TaskContext = { hasScreenshot };
  
  // 统一走 TaskOrchestrator
  const result = await this.orchestrator.process(message, context);
  
  return {
    id: request.id,
    result: {
      response: result.response,
      tier: result.tier,
      executionTime: result.executionTime,
    },
  };
}
```

### P0-2: 修复意图缓存竞态条件

**问题**: `IntentCache` 的 `evictIfNeeded()` 方法在并发场景下可能导致缓存大小计算不准确。

**涉及文件**:
- `src/core/IntentCache.ts`

**现状**:
```typescript
private evictIfNeeded(): void {
  const totalSize = this.directMatchCache.size + this.classificationCache.size; // ❌ 快照值
  if (totalSize > this.maxCacheSize) {
    // 驱逐逻辑...
  }
}
```

**修复方案**:
```typescript
private evictIfNeeded(): void {
  while (this.currentCacheSize > this.maxCacheSize && this.accessOrder.length > 0) {
    const oldest = this.accessOrder.shift();
    if (oldest) {
      this.directMatchCache.delete(oldest);
      this.classificationCache.delete(oldest);
    }
  }
}

private get currentCacheSize(): number {
  return this.directMatchCache.size + this.classificationCache.size;
}
```

### P0-3: 补充 TaskResult 中的模型名称

**问题**: `TaskResult` 缺少 `model` 字段，无法追踪使用了哪个 AI 模型。

**涉及文件**:
- `src/core/TaskOrchestrator.ts`
- `src/types/orchestrator.ts`

**现状**:
```typescript
return {
  success: results.every(r => r.success),
  response: summary,
  tier: 'smart',
  executionTime: Date.now() - startTime,
  // ❌ 缺少 model 字段
};
```

**修复方案**:
1. 在 `TaskResult` 接口中添加 `model?: string` 字段
2. 从 `AIRouter` 获取当前使用的模型名

```typescript
// types/orchestrator.ts
export interface TaskResult {
  success: boolean;
  response: string;
  tier: 'direct' | 'fast' | 'vision' | 'smart';
  executionTime: number;
  model?: string; // 新增
}

// TaskOrchestrator.ts
return {
  success: true,
  response: summary,
  tier: 'smart',
  executionTime: Date.now() - startTime,
  model: this.smartEngine.modelName, // 新增
};
```

### P1-2: 优化工具 Schema 生成

**问题**: 当前工具定义将所有 action 的参数聚合到一个 schema，导致 AI 可能传入不匹配的参数组合。

**涉及文件**:
- `src/core/ToolRegistry.ts`

**现状**:
```typescript
get parameters(): Record<string, any> {
  // 聚合所有 action 的参数 (去重)
  const allParams: Record<string, any> = {};
  for (const action of this.actions) {
    Object.assign(allParams, action.parameters);
  }
  return allParams;
}
```

**修复方案 A - 每个 action 一个 tool**:
```typescript
/**
 * 生成 OpenAI Function Calling 工具列表（每个 action 一个 tool）
 */
toOpenAITools(): OpenAITool[] {
  return this.actions.map(action => ({
    type: 'function',
    function: {
      name: `${this.id}_${action.id}`,
      description: `${this.name} - ${action.description}`,
      parameters: {
        type: 'object',
        properties: action.parameters,
        required: action.requiredParams || [],
      },
    },
  }));
}
```

**修复方案 B - 使用 oneOf 分支 schema**:
```typescript
get parameters(): Record<string, any> {
  const actionSchemas = this.actions.map(action => ({
    type: 'object',
    properties: {
      action: { const: action.id },
      ...action.parameters,
    },
    required: ['action', ...(action.requiredParams || [])],
  }));

  return {
    oneOf: actionSchemas,
  };
}
```

**问题**: `executeFastLayer` 没有传递对话历史，导致 AI 无法理解上下文。

**涉及文件**:
- `src/core/TaskOrchestrator.ts`
- `src/core/ContextManager.ts`

**修复方案**:
```typescript
// TaskOrchestrator.ts - executeFastLayer
private async executeFastLayer(input: string, context: TaskContext): Promise<TaskResult> {
  // 获取历史上下文
  const history = this.contextManager.getRecentHistory(5);
  
  const messages = [
    ...history.map(h => ({ role: h.role, content: h.content })),
    { role: 'user' as const, content: input },
  ];
  
  const response = await this.fastEngine.chat({
    messages,
    tools: this.toolExecutor.getAvailableTools(),
  });
  
  // 保存到历史
  this.contextManager.addMessage({ role: 'user', content: input });
  this.contextManager.addMessage({ role: 'assistant', content: response.content });
  
  // ... 后续处理
}
```

### P1-4: 增强 JSON 解析容错

**问题**: `TaskPlanner` 从 AI 响应中提取 JSON 时，容错能力不足。

**涉及文件**:
- `src/core/TaskPlanner.ts`

**修复方案**:
```typescript
/**
 * 从 AI 响应中提取 JSON（增强容错）
 */
private extractJSON(text: string): string | null {
  // 1. 尝试 markdown 代码块
  const codeBlockMatch = text.match(/```(?:json)?\s*([\s\S]*?)\s*```/);
  if (codeBlockMatch) {
    return codeBlockMatch[1].trim();
  }
  
  // 2. 尝试直接匹配 JSON 对象
  const jsonMatch = text.match(/\{[\s\S]*\}/);
  if (jsonMatch) {
    try {
      JSON.parse(jsonMatch[0]);
      return jsonMatch[0];
    } catch {
      // 继续尝试其他方法
    }
  }
  
  // 3. 尝试修复常见问题
  let cleaned = text
    .replace(/,\s*}/g, '}')  // 移除尾随逗号
    .replace(/,\s*]/g, ']')  // 移除数组尾随逗号
    .replace(/'/g, '"');     // 单引号转双引号
  
  const cleanedMatch = cleaned.match(/\{[\s\S]*\}/);
  if (cleanedMatch) {
    try {
      JSON.parse(cleanedMatch[0]);
      return cleanedMatch[0];
    } catch {
      // 解析失败
    }
  }
  
  return null;
}
```

// tests/IntentAnalyzer.test.ts
import { describe, it, expect, beforeEach, vi } from 'vitest';
import { IntentAnalyzer, TaskType } from '../src/core/IntentAnalyzer';

// Mock AdapterRegistry
const mockRegistry = {
  getAll: vi.fn(() => []),
  get: vi.fn(),
};

describe('IntentAnalyzer', () => {
  let analyzer: IntentAnalyzer;

  beforeEach(() => {
    analyzer = new IntentAnalyzer(mockRegistry as any);
  });

  describe('matchDirectTool', () => {
    it('should match volume increase', async () => {
      const result = await analyzer.analyze('调高音量', {});
      expect(result.taskType).toBe(TaskType.Simple);
      expect(result.directToolCall).toEqual({
        tool: 'audio',
        action: 'increase',
        params: {},
      });
    });

    it('should match app launch with name', async () => {
      const result = await analyzer.analyze('打开 Chrome', {});
      expect(result.directToolCall?.params.name).toBe('Chrome');
    });

    it('should return null for unmatched input', async () => {
      const result = await analyzer.analyze('今天天气怎么样', {});
      expect(result.directToolCall).toBeUndefined();

  describe('requiresVision', () => {
    it('should detect vision keywords', async () => {
      const result = await analyzer.analyze('点击屏幕上的按钮', {});

    });

    it('should not detect vision for simple commands', async () => {
      const result = await analyzer.analyze('调高音量', {});
      expect(result.taskType).toBe(TaskType.Simple);
