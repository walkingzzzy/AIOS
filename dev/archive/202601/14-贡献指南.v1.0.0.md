## 开发环境

### 系统要求

| 平台 | 最低版本 |
|------|---------|
| Windows | Windows 10 |
| macOS | macOS 13 (Ventura)+ |
| Linux | Ubuntu 22.04+ / Fedora 38+ |

### 工具要求

```bash
# Node.js 20+
node --version  # v20.x.x

# pnpm
npm install -g pnpm
pnpm --version

# Git
git --version
```

### 克隆项目

```bash
git clone https://github.com/aios-protocol/aios.git
cd aios
pnpm install
```

## 代码规范

### TypeScript 规范

- 使用 TypeScript 5.x 严格模式
- 所有公共 API 必须有 JSDoc 注释
- 导出类型定义

```typescript
/**
 * 适配器调用结果
 */
export interface AdapterResult<T = unknown> {
  success: boolean;
  data?: T;
  error?: string;
}
```

### 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 文件 | kebab-case | `audio-adapter.ts` |
| 类 | PascalCase | `AudioAdapter` |
| 函数 | camelCase | `setVolume()` |
| 常量 | UPPER_SNAKE | `MAX_VOLUME` |
| 接口 | PascalCase + I 前缀可选 | `AdapterResult` |

### 提交规范

使用 [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: 添加音量控制适配器
fix: 修复 Linux 亮度控制权限问题
docs: 更新适配器开发指南
test: 添加 AudioAdapter 单元测试
refactor: 重构权限管理器
```

## 适配器开发

### 创建新适配器

1. 在 `packages/adapters/` 下创建目录
2. 实现 `BaseAdapter` 接口
3. 创建 `tool.aios.yaml` 能力描述
4. 添加单元测试
5. 在 `AdapterRegistry` 中注册

详见 [08-适配器开发指南](./08-适配器开发指南.md)

## Pull Request

1. Fork 项目
2. 创建特性分支: `git checkout -b feat/my-feature`
3. 提交更改: `git commit -m 'feat: 添加新功能'`
4. 推送分支: `git push origin feat/my-feature`
