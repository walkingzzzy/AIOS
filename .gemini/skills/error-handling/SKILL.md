# 错误处理

## 错误分类

| 类型 | 描述 | 处理策略 |
|------|------|---------|
| 可恢复错误 | 临时故障，可重试 | 重试 + 指数退避 |
| 不可恢复错误 | 逻辑错误，无法恢复 | 快速失败 + 记录 |
| 边界错误 | 输入无效 | 验证 + 用户反馈 |
| 系统错误 | 环境问题 | 降级 + 告警 |

---

## 错误处理原则

### 强制执行
```
✅ 显式处理所有错误
✅ 在边界处验证输入
✅ 错误信息包含上下文
✅ 区分用户错误和系统错误
✅ 使用类型系统表达错误
```

### 禁止行为
```
❌ 静默吞掉错误
❌ 使用通用错误消息
❌ 在内部传播用户输入错误
❌ 错误处理中产生新错误
```

## 日志规范

### 日志级别

| 级别 | 用途 | 示例 |
|------|------|------|
| ERROR | 需要立即关注的错误 | 数据库连接失败 |
| WARN | 潜在问题，可继续运行 | 重试成功 |
| INFO | 关键业务事件 | 用户登录 |
| DEBUG | 调试信息 | 函数参数 |

### 日志内容要求
```
✅ 时间戳
✅ 错误类型和消息
✅ 调用栈/上下文
✅ 相关 ID（请求ID、用户ID）
❌ 敏感信息（密码、密钥）
```

## 重试策略

### 指数退避
```
第1次重试: 1秒
第2次重试: 2秒
第3次重试: 4秒
...
最大重试: 5次
最大等待: 30秒
```

### 重试条件
- ✅ 网络超时
- ✅ 服务暂时不可用
- ✅ 限流响应
- ❌ 认证失败
- ❌ 参数错误

---

## 优雅降级

### 降级策略
1. **功能降级**：禁用非核心功能
2. **数据降级**：返回缓存/默认值
3. **体验降级**：简化 UI/响应

### 熔断器模式
```
正常 → 检测到故障 → 打开熔断 → 等待恢复 → 半开状态 → 恢复正常
```
