# 文档编写标准

## 文档类型

| 类型 | 目的 | 受众 |
|------|------|------|
| README | 项目入口，快速上手 | 所有开发者 |
| API 文档 | 接口说明 | API 使用者 |
| 架构文档 | 设计决策 | 团队成员 |
| 代码注释 | 实现说明 | 维护者 |
| CHANGELOG | 版本变更 | 用户/开发者 |

---

## README 模板

```markdown
# 项目名称

简短描述（一句话）

## 功能特性
- 功能1
- 功能2

## 快速开始

### 安装
[安装命令]

### 使用
[基本使用示例]

## 配置
[配置说明]

## 贡献指南
[如何贡献]

## 许可证
[许可证信息]
```

## API 文档规范

### 必须包含
```
✅ 接口路径和方法
✅ 请求参数（类型、必选、描述）
✅ 响应格式和示例
✅ 错误码和含义
✅ 使用示例
```

### 示例格式
```markdown
## 获取用户信息

**请求**: `GET /api/users/{id}`

**参数**:
| 名称 | 类型 | 必选 | 描述 |
|------|------|------|------|
| id | string | 是 | 用户ID |

**响应**:
- 200: 成功
- 404: 用户不存在

**示例**:
[代码示例]
```

## 代码注释规范

### 必须注释
- 公共 API（函数、类、方法）
- 复杂算法
- 非显而易见的业务逻辑
- TODO 和 FIXME

### 注释格式
```typescript
/**
 * 计算用户订单总金额
 * 
 * @param userId - 用户唯一标识
 * @param options - 计算选项
 * @returns 订单总金额（单位：分）
 * @throws {UserNotFoundError} 用户不存在时抛出
 */
```

### 禁止
```
❌ 重复代码的注释
❌ 过时的注释
❌ "修改记录"式注释
❌ 被注释掉的代码
```

## CHANGELOG 规范

### 格式（Keep a Changelog）
```markdown
## [版本号] - 日期

### Added（新增）
- 新功能描述

### Changed（变更）
- 变更描述

### Fixed（修复）
- 修复描述

### Removed（移除）
- 移除描述
```

## 文档维护原则

- ✅ 文档与代码同步更新
- ✅ 使用简体中文（遵循项目规范）
- ✅ 提供可运行的示例
- ✅ 定期审查过时文档
