# 调试

## 调试方法论

### 科学调试法
```
1. 复现：稳定复现问题
2. 定位：缩小问题范围
3. 分析：理解根本原因
4. 修复：解决问题
5. 验证：确认修复有效
6. 预防：防止再次发生
```

---

## 问题定义（5W1H）

| 问题 | 描述 |
|------|------|
| **What** | 具体问题是什么？ |
| **When** | 什么时候发生？ |
| **Where** | 在哪里发生？ |
| **Who** | 影响谁？ |
| **Why** | 可能的原因？ |
| **How** | 如何复现？ |

## 信息收集

### 必须收集
```
✅ 错误消息和堆栈跟踪
✅ 相关日志
✅ 环境信息（版本、配置）
✅ 复现步骤
✅ 最近的变更
```

### 收集工具
```
日志：tail -f logs/app.log
错误：grep -i "error" logs/app.log
性能：top / htop / profiler
网络：curl / network inspector
```

## 定位技术

### 二分法
```
1. 找到问题发生的范围
2. 将范围一分为二
3. 确定问题在哪一半
4. 重复直到找到精确位置
```

### 最近变更法
```
1. 检查最近的代码变更
2. 检查最近的配置变更
3. 检查最近的环境变更
4. 逐一回滚验证
```

### 最小复现法
```
1. 创建最小测试用例
2. 逐步移除不相关代码
3. 直到找到最小可复现的代码
```

## 常见问题模式

| 现象 | 可能原因 |
|------|---------|
| 偶发性失败 | 竞态条件、资源泄露 |
| 性能下降 | 内存泄露、N+1 查询 |
| 空指针异常 | 未处理的空值 |
| 数据不一致 | 并发写入、缓存失效 |
| 超时错误 | 死锁、慢查询 |

## 调试心态

- ✅ 保持冷静，系统思考
- ✅ 质疑假设，验证每一步
- ✅ 记录过程，避免重复尝试
- ❌ 不要猜测，用证据说话
- ❌ 不要一次改多处
