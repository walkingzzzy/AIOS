## 概述

**AIOS Protocol** 是 AI 系统控制的开放标准 —— 定义 AI 如何描述、调用和安全执行系统控制能力。

> **核心设计原则**：标准化「接口」，而非「实现」—— 与 MCP 互补，专注系统控制领域。

本文档作为协议规范的索引，各部分详细规范请参阅对应文档。

## 合规性要求

### 必须实现 (MUST)

1. JSON-RPC 2.0 消息格式
2. `aios/capability.list` 和 `aios/capability.invoke` 方法
3. 五级权限模型（public/low/medium/high/critical）
4. 标准错误码（-32001 ~ -32199）

### 应该实现 (SHOULD)

1. 至少一种传输方式 (推荐 Unix Socket)
2. 会话管理 (`aios/session.*`)
3. 权限管理 (`aios/permission.*`)
4. 审计日志

### 可选实现 (MAY)

1. 流式响应
2. 批量操作
3. MCP/A2A 互操作
4. WASM 沙箱

## 术语定义

| 术语 | 定义 |
|------|------|
| **Adapter** | 连接 AIOS 与目标软件的桥梁 |
| **Capability** | 适配器暴露的具体控制功能 |
| **Intent** | 用户自然语言表达的需求 |
| **Permission** | 控制 AI 可执行操作的机制 |
| **Token** | 权限授予的细粒度凭证 |

→ 完整术语定义见 [核心概念](01-CoreConcepts.md)
