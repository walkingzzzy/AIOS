## 一、适配器开发原则

### 1.1 设计原则

| 原则 | 说明 |
|------|------|
| **单一职责** | 每个适配器只负责一个软件/功能域 |
| **最小权限** | 只请求必要的权限 |
| **优雅降级** | 部分功能失败不影响整体 |
| **用户友好** | 返回有帮助的错误信息 |
| **可测试性** | 易于编写单元测试和集成测试 |
| **语言无关** | 可使用任何支持的语言实现 |

### 1.2 语言选择指南

AIOS 支持多种编程语言，选择适合您场景的语言：

| 语言 | 适用场景 | SDK |
|------|---------|-----|
| **Python** | 快速开发、AI集成、脚本化 | `aios-sdk` |
| **TypeScript** | Web集成、Node.js生态 | `@aios/sdk` |
| **Go** | 高性能服务、微服务 | `aios-go` |
| **Rust** | 系统级控制、高安全性 | `aios-sdk` (crate) |
| **Java/Kotlin** | 企业应用、Android | `aios-jvm` |

### 1.3 适配器类型选择

| 类型 | 适用场景 | 优先级 |
|------|---------|-------|
| D-Bus | Linux 系统服务 | P1 |
| 原生 API | 软件提供 Python/脚本 API | P1 |
| CLI | 命令行工具 | P2 |
| AT-SPI | GUI 应用无 API | P3 |
| GUI 自动化 | 最后手段 | P4 |

### 1.4 部署方式选择

| 部署方式 | 适用场景 | 传输协议 |
|---------|---------|---------|
| **本地进程** | CLI工具、开发测试 | stdio |
| **本地服务** | 桌面环境、后台服务 | Unix Socket / HTTP |
| **远程服务** | 云端部署、远程访问 | HTTP / WebSocket |
| **Serverless** | 按需扩展、低成本 | HTTP |
| **容器化** | 隔离部署、微服务 | HTTP |

### 1.5 能力设计规范

| 要求 | 说明 |
|------|------|
| 命名清晰 | 全局能力ID使用 `<namespace>.<category>.<action>`，action 部分用动词_名词，如 `system.desktop.set_wallpaper` |
| 参数最小化 | 只包含必要参数，可选参数有默认值 |
| 输出结构化 | 返回结构化数据，便于 AI 理解 |
| 示例完整 | 提供多个 AI 调用示例 |

## 二、性能优化

### 2.1 性能目标

| 指标 | 目标值 | 说明 |
|------|-------|------|
| 初始化时间 | < 500ms | 冷启动到就绪 |
| 普通调用延迟 | < 100ms | P95 延迟 |
| D-Bus 调用延迟 | < 50ms | P95 延迟 |
| 内存占用 | < 50MB | 稳定运行时 |
| 并发处理 | > 100 QPS | 持续负载 |

### 2.2 优化策略

| 策略 | 说明 | 优先级 |
|------|------|-------|
| 连接复用 | 复用 D-Bus 连接和 HTTP 会话 | 高 |
| 延迟初始化 | 按需初始化资源 | 高 |
| 结果缓存 | 缓存不变或慢变化的数据 | 中 |
| 批量操作 | 合并多个操作为批量调用 | 中 |
| 异步执行 | 使用异步 I/O 避免阻塞 | 高 |

### 2.3 缓存策略

| 数据类型 | 缓存时间 | 说明 |
|---------|---------|------|
| 系统信息 | 5 分钟 | 主机名、版本等 |
| 设置值 | 1 分钟 | 可能被用户修改 |
| 网络状态 | 10 秒 | 变化较快 |
| 应用列表 | 5 分钟 | 安装/卸载不频繁 |

### 2.4 D-Bus 优化

| 优化点 | 说明 |
|-------|------|
| 使用 GetAll | 一次获取所有属性，而非多次调用 |
| 订阅信号 | 使用信号而非轮询检测变化 |
| 批量调用 | 合并多个 D-Bus 调用 |
| 连接池 | 复用 Session/System Bus 连接 |

## 三、错误处理

### 3.1 错误处理原则

| 原则 | 说明 |
|------|------|
| 快速失败 | 尽早发现并报告错误 |
| 详细日志 | 记录足够的上下文信息 |
| 优雅降级 | 部分功能失败不影响整体 |
| 用户友好 | 返回有帮助的错误信息 |
| 安全优先 | 不暴露内部实现细节 |

### 3.2 错误处理流程

```
请求到达
    ↓
参数验证 → 失败 → 返回 -32108 (验证失败)
    ↓ 成功
权限检查 → 失败 → 返回 -32001 (权限拒绝)
    ↓ 成功
执行操作 → 失败 → 根据错误类型返回对应错误码
    ↓ 成功
返回结果
```

### 3.3 错误码使用规范

| 错误码范围 | 类别 | 示例 |
|-----------|------|------|
| -32001 ~ -32010 | 协议错误 | 权限拒绝、版本不兼容 |
| -32100 ~ -32199 | 业务错误 | 能力不存在、任务失败 |

→ 详见 [错误码规范](../protocol/AIOS-Protocol-ErrorCodes.md)

### 3.4 错误信息规范

| 字段 | 必需 | 说明 |
|------|------|------|
| code | ✅ | 错误码 |
| message | ✅ | 用户可读的错误描述 |
| error_id | ✅ | 唯一错误标识 (如 AIOS-PERM-001) |
| details | ❌ | 详细错误信息 (调试用) |
| retryable | ❌ | 是否可重试 |
| trace_id | ❌ | 追踪 ID (用于日志关联) |

## 四、安全规范

### 4.1 输入验证清单

| 检查项 | 说明 |
|-------|------|
| JSON Schema 验证 | 所有输入都经过 Schema 验证 |
| 路径遍历检查 | 文件路径参数检查 `../` |
| 命令注入检查 | 命令参数进行转义 |
| 长度限制 | 限制输入字符串长度 |
| 类型检查 | 验证参数类型正确 |

### 4.2 权限声明规范

| 要求 | 说明 |
|------|------|
| 完整声明 | 所有能力都声明所需权限 |
| 最小权限 | 只请求必要的权限 |
| 范围限定 | 尽可能精确的 scope |
| 时效最短 | 优先使用 once/task |
| 原因说明 | 清晰的权限用途说明 |

### 4.3 数据处理规范

| 要求 | 说明 |
|------|------|
| 敏感数据不记录 | 密码、令牌等不写入日志 |
| 错误信息脱敏 | 不暴露内部实现细节 |
| 临时文件清理 | 使用后删除临时文件 |
| 内存清理 | 敏感数据使用后清零 |

### 4.4 安全审计清单

**开发阶段**：
| ID | 检查项 | 严重性 |
|----|-------|-------|
| SEC-DEV-001 | 所有输入都经过验证 | 高 |
| SEC-DEV-002 | 文件路径检查路径遍历 | 严重 |
| SEC-DEV-003 | 命令执行参数进行转义 | 严重 |
| SEC-DEV-010 | 所有能力都声明所需权限 | 高 |
| SEC-DEV-011 | 执行前验证权限令牌 | 严重 |
| SEC-DEV-020 | 敏感数据不记录到日志 | 高 |
| SEC-DEV-021 | 错误信息不暴露内部详情 | 中 |

**发布阶段**：
| ID | 检查项 | 工具 |
|----|-------|------|
| SEC-REL-001 | 所有依赖无已知漏洞 | pip-audit / npm audit |
| SEC-REL-002 | 依赖版本固定 | lock file |
| SEC-REL-010 | 代码通过安全审查 | 人工审查 |
| SEC-REL-011 | 无硬编码凭证 | git-secrets |

## 五、测试规范

### 5.1 测试分类

| 类型 | 目的 | 覆盖率要求 |
|------|------|----------|
| 单元测试 | 测试单个函数/方法 | > 80% |
| 集成测试 | 测试组件交互 | 关键路径 |
| 协议兼容测试 | 验证协议符合性 | 100% |
| 性能测试 | 测试性能指标 | 关键能力 |

### 5.2 单元测试要求

| 要求 | 说明 |
|------|------|
| 正常情况 | 测试正常输入和预期输出 |
| 边界情况 | 测试边界值和极端情况 |
| 错误情况 | 测试各种错误场景 |
| 权限检查 | 测试权限拒绝场景 |
| Mock 依赖 | 使用 Mock 隔离外部依赖 |

### 5.3 协议兼容性测试

| 测试项 | 说明 |
|-------|------|
| tool.aios.yaml 格式 | 验证工具描述文件格式正确 |
| 请求格式 | 验证请求符合 JSON-RPC 2.0 |
| 响应格式 | 验证响应格式正确 |
| 错误格式 | 验证错误响应格式正确 |
| 未知字段处理 | 验证未知字段被正确忽略 |

### 5.4 性能测试指标

| 指标 | 测量方法 | 目标 |
|------|---------|------|
| 延迟 P50 | 中位数延迟 | < 50ms |
| 延迟 P95 | 95 分位延迟 | < 100ms |
| 延迟 P99 | 99 分位延迟 | < 200ms |
| 吞吐量 | 每秒请求数 | > 100 QPS |
| 错误率 | 失败请求比例 | < 1% |

## 七、发布规范

### 7.1 版本号规范

遵循语义化版本 (SemVer)：

| 版本类型 | 格式 | 说明 |
|---------|------|------|
| 主版本 | X.0.0 | 不兼容的 API 变更 |
| 次版本 | 0.X.0 | 向后兼容的功能新增 |
| 补丁版本 | 0.0.X | 向后兼容的问题修复 |

### 7.2 发布清单

| 检查项 | 说明 |
|-------|------|
| 版本号更新 | 更新 tool.aios.yaml 中的版本 |
| 变更日志 | 更新 CHANGELOG |
| 测试通过 | 所有测试通过 |
| 安全审计 | 安全检查通过 |
| 文档更新 | 文档与代码同步 |

### 7.3 兼容性要求

| 要求 | 说明 |
|------|------|
| 向后兼容 | 次版本和补丁版本必须向后兼容 |
| 废弃警告 | 废弃功能提前一个次版本警告 |
| 迁移指南 | 主版本升级提供迁移指南 |
