# 16 · 多模态采集与设备能力规范

**版本**: 1.0.0  
**更新日期**: 2026-03-08  
**状态**: P1/P2 核心规格

---

## 1. 目标

本文件定义 AIOS 如何把文本、屏幕、音频、图像、输入与设备状态纳入系统级能力面。  
它补齐“AI 如何感知当前系统状态”这一关键缺口。

## 2. 基本原则

### 2.1 多模态是系统输入，不是应用附件

在 AIOS 中：

- 屏幕不是某个 app 的截图功能
- 麦克风不是某个助手 app 的私有输入
- 输入事件不是自动化脚本的私有资源

它们都属于 `device.*` 与 `shell.*` 的系统级能力。

### 2.2 感知能力默认受限

多模态采集默认不连续开启，必须遵守：

- 最小范围
- 最短时间
- 最少模态
- 最小保留

### 2.3 感知数据必须带来源与 taint

所有多模态输入必须带上：

- 来源设备
- 来源会话
- 采集时间
- 权限状态
- 是否 tainted

## 3. 模态分类

| 模态 | 说明 | 典型 capability |
|------|------|------------------|
| 文本 | 用户输入、剪贴板、结构化内容 | `device.capture.text.*` |
| 屏幕像素 | 截图、屏幕片段、窗口画面 | `device.capture.screen.*` |
| UI 结构 | 窗口树、控件树、可访问性树 | `shell.inspect.ui_tree.*` |
| 音频 | 麦克风输入、系统输出、提示音状态 | `device.capture.audio.*` |
| 图像 / 视频 | 摄像头、相机流、视频帧 | `device.capture.camera.*` |
| 输入事件 | 键盘、鼠标、触控、焦点变化 | `device.capture.input.*` |
| 环境状态 | 电量、网络、蓝牙、地理、传感器 | `device.read.environment.*` |

## 4. 设备能力域

推荐最小能力域：

- `device.capture.screen.window`
- `device.capture.screen.region`
- `device.capture.audio.microphone`
- `device.capture.camera.frame`
- `device.capture.input.focus`
- `device.capture.input.pointer_trace`
- `device.read.environment.power`
- `device.read.environment.network`
- `shell.inspect.window_tree`
- `shell.inspect.ui_tree`

## 5. 采集管线

## 5.1 标准管线

```text
Device / Shell Event
  ↓
aios-deviced
  ↓
normalize + annotate + taint mark
  ↓
policyd capability check
  ↓
sessiond context binding
  ↓
runtimed / agentd consume
  ↓
audit + retention policy
```

## 5.2 标准处理步骤

### 采集

从设备、窗口系统、可访问性树或系统状态采集原始信号。

### 归一化

转成统一对象：

- `screen_frame`
- `ui_tree_snapshot`
- `audio_chunk`
- `camera_frame`
- `input_event_batch`
- `environment_state`

### 标注

添加：

- 时间戳
- 来源设备
- 来源窗口 / workspace
- 用户 / 会话绑定
- 权限状态
- taint 状态

### 消费

交给：

- `agentd` 用于规划
- `sessiond` 用于上下文绑定
- `runtimed` 用于模型推理

### 处理后约束

- 默认不长期保存原始敏感流
- 保存应优先保存摘要、结构化特征或低风险引用

## 6. 权限与审批模型

## 6.1 默认低风险读取

默认可低风险读取的仅限：

- 非敏感环境状态
- 前台窗口标题等低敏元数据
- 非持续性、非原始敏感数据

## 6.2 默认高风险采集

下列能力默认高风险：

- 连续屏幕录制
- 麦克风录音
- 摄像头连续采集
- 全局输入事件采集
- 剪贴板持续监听
- 后台窗口内容采集

这些必须：

- 进入审批链或显式用户授权
- 有可见状态提示
- 可撤销
- 可超时

## 6.3 用户可见性要求

当以下采集开启时，系统必须可见提示：

- 麦克风
- 摄像头
- 屏幕共享 / 屏幕录制
- 输入注入或自动化控制

## 7. 存储与保留策略

### 7.1 原始流

默认不长期保存：

- 麦克风原始流
- 摄像头连续帧
- 长时屏幕视频
- 全量键盘输入流

### 7.2 结构化摘要

允许按策略保存：

- 屏幕摘要
- UI tree snapshot
- 音频转写文本
- 环境状态快照
- 任务相关片段索引

### 7.3 retention 字段

每类模态对象至少要有：

- `retention_class`
- `retention_ttl`
- `storage_scope`
- `redaction_policy`

## 8. 多模态对象模型

### 8.1 `screen_frame`

至少包含：

- `frame_id`
- `window_ref`
- `workspace_ref`
- `timestamp`
- `resolution`
- `visibility_scope`
- `taint_summary`

### 8.2 `ui_tree_snapshot`

至少包含：

- `snapshot_id`
- `window_ref`
- `timestamp`
- `tree_version`
- `source_backend`
- `focus_node`
- `taint_summary`

### 8.3 `audio_chunk`

至少包含：

- `chunk_id`
- `source_device`
- `timestamp`
- `duration_ms`
- `channel_layout`
- `transcript_ref`
- `taint_summary`

### 8.4 `input_event_batch`

至少包含：

- `batch_id`
- `source_device`
- `timestamp_range`
- `event_types`
- `window_ref`
- `focus_ref`
- `taint_summary`

## 9. AI 使用多模态输入的规则

AI 消费多模态输入时必须遵守：

- 先看结构化摘要，再决定是否需要原始流
- 优先前台窗口，不默认读取后台窗口
- 优先 UI tree，不默认依赖像素 OCR
- 优先短时采样，不默认持续监听
- 高敏感数据不得默认进入 long-term memory

## 10. 与自动化的关系

多模态感知与自动化控制必须严格分离：

- 感知是读取
- 自动化是写入 / 控制

例如：

- 读取当前前台窗口结构 → 低 / 中风险
- 向该窗口注入键盘输入 → 高风险

因此：

- `device.capture.*` 与 `compat.ui.*` 不得被混为一谈
- 读取屏幕不等于有资格点击屏幕

## 11. 失败与降级

若多模态管线部分不可用，AIOS 应优先降级为：

- `ui_tree` 不可用 → `screen_frame + OCR`
- 麦克风不可用 → 文本输入
- 摄像头不可用 → 禁用视觉场景理解
- GPU 不足 → 降级到低频采样或本地 CPU 分析

降级必须可解释，并进入审计链。

## 12. v1 最低要求

- 至少支持屏幕、音频、输入三类中的两类
- 至少支持一个 `ui_tree` 或等价窗口结构来源
- 多模态对象带来源与 taint 标记
- 高风险连续采集具备显式授权与可视提示
- 多模态摘要可绑定到会话与任务
