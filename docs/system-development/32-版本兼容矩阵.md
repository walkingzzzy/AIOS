# 32 · 版本兼容矩阵

**版本**: 1.1.0  
**更新日期**: 2026-03-14  
**状态**: `P7-REL-012` frozen baseline

---

## 1. 目标

本文冻结 AIOS 当前平台交付链、runtime profile、observability schema 与 task metadata 的兼容关系。

它回答三个问题：

1. 哪些 profile / artifact 组合允许一起发布
2. 哪些 schema / contract 版本是当前仓库基线
3. 哪些混搭必须显式禁止

## 2. 兼容性原则

1. image 平台 profile、`updated` platform profile、runtime profile 与 hardware profile 必须按同一平台组合发布。
2. `qemu-x86_64` 只代表仓库/QEMU 验证基线，不能与 Tier 1 硬件支持结论混用。
3. 任何带 `schema_version` 的 observability payload 都必须符合当前支持版本集合。
4. `docs/system-development/21-完整任务清单.md` 与 `22-执行任务看板.md` 更新后，必须重新生成 `aios/tasks/*.yaml`。
5. 新增平台或 profile 路径变更时，必须同步更新 [30-支持矩阵与已知限制.md](30-支持矩阵与已知限制.md) 与发布说明模板。
6. 正式 Tier 1 机器可以在平台组合之上增加 machine-specific hardware profile，但必须显式绑定到唯一 platform combo。

## 3. 当前发布组合矩阵

| 发布组合 | image / platform profile | `updated` platform profile | runtime profile | hardware profile | boot backend | 当前结论 |
|----------|--------------------------|----------------------------|-----------------|------------------|--------------|----------|
| `qemu-x86_64` 仓库/QEMU 基线 | `aios/image/profiles/qemu-x86_64/profile.yaml` | `/usr/share/aios/updated/platforms/qemu-x86_64/profile.yaml` | `default-local` + `default-route` | `qemu-x86_64` | `state-file` | 允许作为 repo/QEMU 验证组合，不允许作为 Tier 1 硬件支持结论 |
| `generic-x86_64-uefi` 平台交付组合 | `aios/image/platforms/generic-x86_64-uefi/profile.yaml` | `/usr/share/aios/updated/platforms/generic-x86_64-uefi/profile.yaml` | `/etc/aios/runtime/default-runtime-profile.yaml` + `default-route` | `generic-x86_64-uefi` | `firmware` | 允许导出 platform media / bring-up kit；仍需真实硬件验证报告才能进入正式支持 |
| `nvidia-jetson-orin-agx` 平台交付组合 | `aios/image/platforms/nvidia-jetson-orin-agx/profile.yaml` | `/usr/share/aios/updated/platforms/nvidia-jetson-orin-agx/profile.yaml` | `/usr/share/aios/runtime/platforms/nvidia-jetson-orin-agx/default-runtime-profile.yaml` + `default-route` | `nvidia-jetson-orin-agx` | `firmware` | 允许导出 bring-up kit 与平台资产；仍需真实硬件验证报告才能进入正式支持 |

当前正式机器绑定：

- `framework-laptop-13-amd-7040` -> `generic-x86_64-uefi` 平台交付组合
- `nvidia-jetson-orin-agx` -> `nvidia-jetson-orin-agx` 平台交付组合

## 4. Schema / Contract 基线

| 域 | 当前版本基线 | 生产方 | 消费方 | 兼容规则 |
|----|--------------|--------|--------|----------|
| task metadata | `1.0.0` | `scripts/sync-aios-task-metadata.py`、`21/22` 规划文档 | `aios/tasks/master-task-list.yaml`、`aios/tasks/execution-board.yaml`、image 内镜像副本 | 规划文档改动后必须重新 sync；不同步时视为不兼容 |
| observability payload | `2026-03-13` | `audit-event`、`trace-event`、`diagnostic-bundle`、`health-event`、`recovery-evidence` exporter | schema validator、report builder、release evidence consumer | 新写入 payload 应带当前版本；若 payload 带版本但不是当前支持集合，则视为不兼容 |
| runtime worker contract | `runtime-worker-v1` | runtime profile、managed worker launcher、reference worker | `runtimed`、managed worker health / readiness gate | worker 返回的 contract 与 backend id 必须匹配；Jetson 组合尤其不能混用其他 contract |
| route profile | `default-route` | runtime / route profile 资产 | route resolver、runtime path | 当前三套平台组合均以 `default-route` 为 route baseline，变更时需同步发布说明与兼容矩阵 |
| validation / release reports | 同仓库快照生成 | `test-system-delivery-validation.py`、`check-release-gate.py`、`run-aios-ci-local.py` | release process、operator handoff | report 与 evidence index 应来自同一次仓库快照，不能跨版本拼接 |

## 5. 当前允许的验证证据组合

| 组合 | 最低证据 | 是否足够做硬件支持宣称 |
|------|----------|------------------------|
| `qemu-x86_64` | QEMU bring-up / recovery / installer-media / cross-reboot + `system-delivery-validation` | 否 |
| `generic-x86_64-uefi` | platform media export + platform profile smoke + firmware backend smoke | 否 |
| `generic-x86_64-uefi` + hardware report | 上述证据 + `hardware-validation-report.md` + `hardware-validation-evidence.json` | 是，前提是报告通过 |
| `nvidia-jetson-orin-agx` | platform media export + bundled runtime assets + platform profile smoke | 否 |
| `nvidia-jetson-orin-agx` + hardware report | 上述证据 + `hardware-validation-report.md` + `hardware-validation-evidence.json` | 是，前提是报告通过 |

## 6. 明确禁止的混搭

- 不得把 `qemu-x86_64` 的验证结果写成 `generic-x86_64-uefi` 或 `nvidia-jetson-orin-agx` 的硬件支持结论。
- 不得为 `generic-x86_64-uefi` 平台介质写入非 `generic-x86_64-uefi` 的 `updated` platform profile。
- 不得为 `nvidia-jetson-orin-agx` 平台介质回退到默认 x86_64 runtime profile，再宣称 Jetson managed worker 路径成立。
- 任何带 `schema_version` 的 observability payload，若不是 `2026-03-13`，当前仓库不应把它作为正式 release 证据。
- 修改 `21-完整任务清单.md` 或 `22-执行任务看板.md` 后，若未重新生成 `aios/tasks/*.yaml`，则视为任务元数据不兼容。
- 不得把来自不同仓库快照的 validation report、release gate report 与 hardware evidence index 拼成同一版 release 说明。

## 7. 变更控制

以下任一项发生变化时，必须同步更新本文：

- 新增平台 profile 或更改 `platform_id`
- 更改 `updated` platform profile 路径
- 更改 runtime profile 路径或 worker contract
- 提升 observability `schema_version`
- 提升 task metadata `schema_version`
- 引入新的 release report / evidence index 契约

## 8. 当前结论

截至 2026-03-14，AIOS 当前兼容矩阵的核心结论是：

- `qemu-x86_64` 是仓库/QEMU 验证组合
- `generic-x86_64-uefi` 与 `nvidia-jetson-orin-agx` 是平台交付组合
- observability payload 版本基线是 `2026-03-13`
- task metadata 版本基线是 `1.0.0`
- 任何超出上述组合的支持宣称，都必须先补 profile、文档、evidence 与 release gate 同步
