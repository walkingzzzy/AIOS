# 33 · 开发者预览发布标准

**版本**: 1.0.0  
**更新日期**: 2026-03-14  
**状态**: Frozen

---

## 1. 目标

本文定义 AIOS `Developer Preview` 的最低发布门槛。

它服务于：

- 系统开发者试用
- 平台 bring-up 与验证
- provider / shell / runtime 集成回归

它不代表：

- 面向终端用户的大规模发布
- 全平台、全硬件的稳定支持
- 完整 SLA 或兼容性承诺

## 2. 必须满足的发布条件

### 2.1 治理与文档

- 已冻结 [30-支持矩阵与已知限制](30-支持矩阵与已知限制.md)
- 已冻结 [31-Release Notes 模板](31-Release-Notes-模板.md)
- 已冻结 [32-版本兼容矩阵](32-版本兼容矩阵.md)
- 当前 release 必须附带已知限制、验证证据与回退说明

### 2.2 控制面与策略

- `agentd` / `sessiond` / `policyd` / `runtimed` / `updated` / `deviced` 必须可编译
- core workspace 单测必须通过
- `scripts/test-ipc-smoke.py` 必须通过
- 高风险 capability 必须具备 approval + audit + execution token 闭环
- prompt injection baseline 与 taint propagation 必须已启用

### 2.3 Shell 与交互

- `launcher`
- `notification-center`
- `task-surface`
- `approval-panel`
- `recovery-surface`

以上 shell surface 至少要有 `prototype/client/panel` 三级产物中的有效实现，并通过最小 smoke：

- `scripts/test-shell-panels-smoke.py`
- `scripts/test-shell-live-smoke.py`
- `scripts/test-shellctl-smoke.py`

### 2.4 Provider / Runtime

- provider registry resolution 必须可用
- `system-files-provider`、`device-metadata-provider`、`runtime-local-inference-provider` 至少要有 smoke 证据
- compat runtime smoke 必须通过
- runtime fallback / managed worker / budget smoke 必须通过

### 2.5 镜像、更新与恢复

- delivery bundle 可生成
- QEMU preflight / bring-up / recovery 至少通过一轮
- `updated` health / restart / platform profile smoke 必须通过
- recovery bundle 与 operator runbook 必须可引用

## 3. 允许存在的限制

开发者预览阶段允许以下限制存在，但必须在支持矩阵或 release notes 中显式声明：

- Tier 1 实机证据尚未覆盖所有目标平台
- 正式 compositor / panel embedding 仍处于原型到正式实现过渡阶段
- 部分 provider 仍是 skeleton / reference backend
- 硬件加速、NPU、可信远端执行仍为条件能力

## 4. 不得缺失的产物

- release notes
- 支持矩阵与已知限制
- 版本兼容矩阵
- validation / cross-service / release gate 报告
- upgrade / recovery runbook

## 5. 当前结论

截至 2026-03-14，当前仓库已基本具备 `Developer Preview` 所需的软件基线，但仍应明确声明：

- 实机硬件覆盖有限
- 安装与 Tier 1 bring-up 仍未完成发行级证据收口
- stable release 口径尚不成立
