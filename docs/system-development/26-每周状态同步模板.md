# 26 · 每周状态同步模板

**版本**: 1.0.0  
**更新日期**: 2026-03-13  
**状态**: 五团队周同步模板

---

## 1. 目的

本模板用于固定 AIOS 五团队的每周状态同步口径，避免根计划、团队计划、任务清单、执行看板和 machine-readable YAML 再次漂移。

适用对象：

- 团队 A：平台交付与更新恢复
- 团队 B：核心控制面与安全治理
- 团队 C：Shell / Portal 交互
- 团队 D：设备 / Provider / Compat 执行
- 团队 E：验证 / 观测 / 发布治理

---

## 2. 使用规则

1. 每周固定更新一次，默认覆盖自然周
2. 每个团队都必须填写完成项、阻塞项、证据与下周目标
3. 若任务状态发生变化，必须同步更新：
   - `docs/system-development/18-开发主计划与任务状态.md`
   - `docs/system-development/21-完整任务清单.md`
   - `docs/system-development/22-执行任务看板.md`
   - `aios/tasks/master-task-list.yaml`
   - `aios/tasks/execution-board.yaml`
4. 若出现高影响接口变更，先补 ADR 或 schema，再进入实现汇报

---

## 3. 周同步头部

```md
# AIOS 周状态同步

- 周期：2026-03-09 ~ 2026-03-15
- 汇总日期：2026-03-15
- 汇总人：<name>
- 总体状态：`On Track` / `At Risk` / `Blocked`
- 本周全局结论：一句话总结当前项目状态
```

---

## 4. 团队状态模板

对每个团队复制一份以下模板：

```md
## 团队 A / 平台交付与更新恢复

- 本周完成项：
  - `<task_id>` <完成内容>
- 当前进行中：
  - `<task_id>` <当前进展>
- 当前阻塞项：
  - <阻塞描述>
- 新增 ADR / 契约需求：
  - <ADR 或 schema 需求；若无填“无”>
- 证据链接：
  - <仓库路径 / 报告路径 / smoke 命令 / artifact 路径>
- 下周目标：
  - `<task_id>` <下周目标>
- 需要其他团队配合：
  - <团队 / 依赖内容；若无填“无”>
```

建议团队顺序：

1. 团队 A / 平台交付与更新恢复
2. 团队 B / 核心控制面与安全治理
3. 团队 C / Shell / Portal 交互
4. 团队 D / 设备 / Provider / Compat 执行
5. 团队 E / 验证 / 观测 / 发布治理

---

## 6. 完成定义

一次有效的周同步至少满足：

- 五个团队都有状态条目
- 每个团队都有可核验的仓库或验证证据
- 至少列出一个当前阻塞项或明确写明“无阻塞”
- 任务状态变化已同步回任务清单和 YAML 导出
