# 34 · 产品预览发布标准

**版本**: 1.0.0  
**更新日期**: 2026-03-14  
**状态**: Frozen

---

## 1. 目标

本文定义 AIOS `Product Preview` 的发布门槛。

它位于开发者预览之上，目标是：

- 面向受控试点环境发布
- 面向少量参考硬件提供可复现交付
- 把“能运行”提升为“可试点、可回退、可排障”

## 2. 必须在开发者预览之上新增满足的条件

### 2.1 硬件与交付

- 至少 1 台正式 Tier 1 机器完成 bring-up 记录
- 至少 1 条安装、升级、回滚、恢复链路有实机证据
- 支持矩阵中声明的平台 profile 必须能映射到实际验证记录

### 2.2 Shell 与用户流

- shell acceptance / stability smoke 必须通过
- launcher / task / approval / notification / recovery 的跨 surface route 必须稳定
- 关键交互流要有 desktop 或 compositor 级证据，而不只是 CLI prototype

### 2.3 安全与治理

- 第一轮安全审计已完成
- 审计中的 `P0/P1` 级问题要么已修复，要么有明确豁免说明
- 高风险 capability 的审计证据必须可被 operator / release artifact 引用

### 2.4 运维与证据

- operator troubleshooting 手册已按当前版本校准
- diagnostic bundle 字段解释与现场分流文档已冻结
- release gate 报告必须可稳定生成

## 3. 仍允许存在的限制

- 支持平台仍可受限于小范围硬件矩阵
- 部分条件能力可继续保持 preview / experimental 标记
- 兼容层 provider 可继续作为过渡方案存在

## 4. 当前结论

截至 2026-03-14，AIOS 尚未达到 `Product Preview` 标准。

主要原因：

- Tier 1 实机 bring-up 证据尚未补齐
- 安装与恢复链路缺少正式硬件覆盖
- release-grade shell/compositor 仍在推进中
