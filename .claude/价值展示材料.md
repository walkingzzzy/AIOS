### 2.4 Demo 1：适配器与能力列表

**演示步骤**：

1. 启动 AIOS Daemon：展示 26 个适配器注册日志
2. 调用 getAdapters 方法：展示适配器列表
3. 展示单个适配器的能力定义（如 AudioAdapter）

**关键讲解点**：

- 18个系统适配器：Audio、Display、Desktop、Power、Apps、SystemInfo、File、Window、Browser、Speech、Notification、Timer、Calculator、Calendar、Weather、Translate、Screenshot、Clipboard
- 8个应用适配器：Spotify、Slack、Discord、Gmail、Outlook、GoogleWorkspace、Notion、Microsoft365
- 每个适配器有明确的能力定义和权限级别

**演示代码**：

```json
{
  "jsonrpc": "2.0",
  "id": "1",
  "method": "getAdapters"
}
```

### 2.5 Demo 2：三层AI协调架构

**演示步骤**：

1. 展示AI引擎配置信息（Fast/Vision/Smart三层）
2. 演示简单任务：直接工具匹配（如"调高音量"）
3. 演示Fast层处理：意图识别 + Function Calling
4. 说明Vision层和Smart层的职责

**关键讲解点**：

- Fast层（gpt-4o-mini）：意图识别、简单任务，<500ms
- Vision层（gemini-2.5-flash）：屏幕分析、元素定位，<3s
- Smart层（claude-sonnet）：任务规划、复杂推理，<5s
- TaskOrchestrator实现三层协调

**演示代码**：

```json
{
  "jsonrpc": "2.0",
  "id": "2",
  "method": "smartChat",
  "params": {
    "message": "调高音量"
  }
}
```

### 2.6 Demo 3：权限管理

**演示步骤**：

1. 展示五级权限模型定义
2. 演示权限检查：checkPermission
3. 演示需要权限的能力调用（如PowerAdapter的关机功能）

## 第三章 对比指标采集表

### 3.1 传统方式 vs AIOS标准化方式

| 指标 | 传统方式 | AIOS标准化方式 | 采集口径 |
|:-----|:---------|:---------------|:---------|
| 需求到可调用接口时间 | 定制开发，周级 | 适配器开发，天级 | 人日/迭代次数 |
| 接口一致性 | 各系统不统一 | JSON-RPC 2.0统一 | 接口规范检查 |
| 权限控制 | 散落/不可追溯 | 五级权限 + 审计 | 合规检查清单 |
| AI集成复杂度 | 需单独对接 | 三层架构内置 | 集成工时 |
| 跨平台支持 | 需分别实现 | 统一API | 平台覆盖率 |
| 可扩展性 | 定制代码 | 适配器插件机制 | 扩展工时 |

## 第四章 典型异议与答法（FAQ）

### 4.1 AIOS和MCP是什么关系？

**答**：AIOS和MCP是互补关系：

- MCP（2024年11月发布）聚焦工具调用域（API/数据库/文件）
- AIOS聚焦系统控制域（操作系统/桌面应用/专业软件）
- AIOS已集成MCP SDK v1.25.1，可调用MCP生态
- AIOS在桥接时叠加权限/审计，形成治理优势

### 4.2 AIOS目前的实现状态如何？

**答**：AIOS v0.1.0已于2026-01-11发布：

- ✅ AIOS Daemon（Node.js守护进程）
- ✅ Electron客户端应用
- ✅ 26个适配器（18系统 + 8应用）
- ✅ 三层AI协调架构
- ✅ 五级权限管理
- ✅ JSON-RPC 2.0通信
- ✅ MCP SDK集成验证

### 4.3 为什么选择Node.js而不是Rust？

**答**：基于实际验证的技术选型：

- Node.js生态丰富，跨平台npm包支持好
- TypeScript提供类型安全
- Electron实现跨平台GUI
- 开发效率高，适合快速迭代
- 已验证的技术栈，风险可控

### 4.4 三层AI架构的优势是什么？

**答**：三层协调而非单选：

- Fast层：简单任务<100ms（本地规则匹配可不经过AI）
- Vision层：屏幕理解，支持GUI自动化
- Smart层：复杂任务规划
- 成本优化：70%简单任务用最便宜的模型
- 响应优化：直接匹配可达<100ms

### 4.5 学校能从中得到什么"可认定成果"？

**答**：

1. 专利/软著：三个应用场景的发明专利 + 软件著作权
2. 课程体系：AIOS标准化开发课程（4-8周模块）
3. 案例库：产教融合案例单元
4. 实验平台：基于AIOS的实训环境
5. 论文素材：技术方案、应用案例、对照实验数据

### 4.6 26个适配器都能用吗？

**答**：

- 18个系统适配器：开箱即用，跨平台支持
- 8个应用适配器：需要OAuth配置后可用
  - Spotify、Slack、Discord：需要开发者账号
  - Gmail、GoogleWorkspace：需要Google Cloud配置
  - Outlook、Microsoft365：需要Azure AD配置
  - Notion：需要Notion Integration
