## 第一章 合作目标与原则

### 1.1 总目标

以 **AIOS v0.1.0 作为"标准开发协议"底座**，与湖北经济学院建立可持续的产教融合合作机制，形成：

1. 一套可教学、可落地、可复用的标准化研发流程
2. 三项应用场景的专利材料与PoC
3. 一批可交付的教学资源与案例库
4. 明确、可执行的知识产权与成果转化方案

### 1.2 AIOS v0.1.0 已具备的合作基础

| 能力 | 状态 | 说明 |
|:-----|:-----|:-----|
| AIOS Daemon | ✅ 可用 | Node.js守护进程 |
| Electron客户端 | ✅ 可用 | 跨平台GUI |
| 26个适配器 | ✅ 可用 | 18系统 + 8应用 |
| 三层AI架构 | ✅ 可用 | Fast/Vision/Smart |
| 五级权限 | ✅ 可用 | PermissionManager |
| JSON-RPC 2.0 | ✅ 可用 | stdio/WebSocket |
| MCP集成 | ✅ 已验证 | SDK v1.25.1 |

### 1.3 合作原则

| 原则 | 说明 |
|:-----|:-----|
| 标准化优先 | 接口/权限/AI协调的规范化交付优先于"单点功能实现" |
| 安全与合规底线 | 五级权限、可审计、可追溯 |
| 分阶段交付 | 以阶段验收替代一次性交付，降低双方风险 |
| 贡献可计量 | 以可追溯的产物作为贡献依据 |

## 第五章 知识产权与成果转化方案

### 5.1 知识产权归属（三套选项）

**方案A：学校主导**

| 项目 | 内容 |
|:-----|:-----|
| 专利/软著权利人 | 学校 |
| 企业获得 | 独占/排他许可 + 优先转化权 |
| 适用情形 | 学校科研成果导向强 |

**方案B：共同持有**

| 项目 | 内容 |
|:-----|:-----|
| 权利人 | 学校 + 企业共同申请 |
| 后续许可 | 需约定一致同意机制 |
| 适用情形 | 双方均投入核心研发 |

**方案C：企业主导**

| 项目 | 内容 |
|:-----|:-----|
| 权利人 | 企业为主 |
| 学校获得 | 科研署名、教学使用权、成果转化分成 |
| 适用情形 | 企业投入更大 |

### 5.2 费用分摊建议

| 费用项目 | 方案A | 方案B | 方案C |
|:---------|:------|:------|:------|
| 专利检索费 | 企业承担 | 各半 | 企业承担 |
| 专利代理费 | 学校承担 | 按比例 | 企业承担 |
| 专利官费 | 学校承担 | 按比例 | 企业承担 |

**费用预算参考**：

| 项目 | 预估费用 |
|:-----|:---------|
| 发明专利（每件） | 代理费8,000-15,000元 + 官费约4,000元 |
| 软著登记（每件） | 300-2,000元 |

### 5.3 AIOS底座的IP处理

建议将AIOS协议/参考实现作为"背景IP"单独处理：

| 资产 | 处理方式 |
|:-----|:---------|
| AIOS协议文本 | 可公开发布 |
| AIOS参考实现（v0.1.0） | 按开源许可或受控开放 |
| 领域扩展适配器 | 作为"前景IP"按上述方案处理 |

### 5.4 学生参与的成果归属

| 序号 | 处理方式 |
|:-----|:---------|
| 1 | 学生在学校安排的实训/毕设中产生的发明创造，通常归学校所有 |
| 2 | 学生作为发明人/设计人的署名权不可剥夺 |
| 3 | 可约定学生参与成果转化收益的分配比例 |
| 4 | 建议学生在参与项目前签署知识产权归属确认书 |

## 第七章 风险评估与应对

| 风险 | 影响 | 可能性 | 应对措施 |
|:-----|:-----|:-------|:---------|
| 业务范围不清 | 专利口径发散 | 高 | 第1阶段完成需求清单签字确认 |
| 数据合规不足 | 案例库无法使用真实数据 | 中 | 采用脱敏/合成数据 |
| IP贡献争议 | 影响申请与转化 | 中 | 以可追溯产物计量贡献 |
| 进度风险 | 无法按学期交付 | 高 | 用阶段门禁拆分里程碑 |
| 学校审批周期长 | 整体进度延迟 | 中 | 提前了解审批流程 |
| 学生成果归属争议 | 影响合作关系 | 中 | 提前签署归属确认书 |
